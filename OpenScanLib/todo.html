Replace this with OSc_Device_GetModelName() and possibly OSc_Device_GetModuleName(). Formatting the name as "name@model" should be the applications responsibility. Same with ensuring that names are unique.
Global OSc_Device_GetName (OSc_Device *device, const char **name)

This API requires OpenScanLib to keep the name string available indefinitely. We should copy to client location.

Return value should be void; we can return "" if device is null and do nothing if destination is null.

Global OSc_Device_GetSettings (OSc_Device *device, OSc_Setting ***settings, size_t *count)
While it makes sense that settings remain valid until the device is closed, the array in which they are returned should be owned by the caller.
Global OSc_GetAllDevices (OSc_Device ***devices, size_t *count)

Return value should be void as we skip any modules and device implementations that have an error. Or we could also return an array of errors.

This API precludes re-enumerating devices. Now that device objects are newly created every time enumeration is performed, we should use a new interface: OSc_EnumerateDevices(OSc_PtrArray *devices).

We will also need support for non-enumerable devices once we add support for them in the device interface.
